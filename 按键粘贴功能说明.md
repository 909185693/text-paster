# 按键粘贴功能说明

## 功能概述

文本快速粘贴工具支持简化模式，可以在保持按住 `Ctrl+Alt` 的情况下，快速切换不同的文本进行粘贴。

## 使用场景

### 场景 1：完整组合触发
1. 按 `Ctrl+Alt+1` → 粘贴文本1
2. 释放所有键后，重新按 `Ctrl+Alt+2` → 粘贴文本2

### 场景 2：简化模式触发（推荐）
1. 按 `Ctrl+Alt+1` → 粘贴文本1
2. **保持按住 Ctrl+Alt 不释放**
3. 按 `2` → 粘贴文本2
4. 按 `3` → 粘贴文本3
5. 释放所有键

## 测试步骤

### 正确的简化模式测试：

1. 启动程序
2. 打开记事本或其他文本编辑器
3. **按住 Ctrl 键不释放**
4. **按住 Alt 键不释放**（此时 Ctrl 和 Alt 都处于按下状态）
5. 按 `1` 键 → 应该粘贴"邮箱地址"
6. **保持 Ctrl 和 Alt 按住**
7. 释放 `1` 键
8. 按 `2` 键 → 应该粘贴"自我介绍"（这是简化模式）
9. **保持 Ctrl 和 Alt 按住**
10. 释放 `2` 键
11. 按 `3` 键 → 应该粘贴"项目经验"（这是简化模式）
12. 释放所有键

### 错误的测试方式（无法触发简化模式）：

❌ 错误：按 `Ctrl+Alt+1`，然后释放所有键，再按 `2`
- 这样无法触发简化模式，因为 Ctrl+Alt 已经被释放了

❌ 错误：只按 `Ctrl+1` 或只按 `Alt+1`
- 这样不会触发任何快捷键，因为必须包含完整的修饰键

## 调试日志解读

当简化模式被触发时，日志中会显示：
```
[TRIGGERED] Hotkey (simplified): ctrl+alt+2
```

当简化模式被跳过时，日志中会显示：
```
[SIMPLIFIED] Skipped: modifiers {('modifier', 'ctrl')} do not contain Ctrl+Alt
```

## 注意事项

1. 简化模式的前提是**之前触发过完整组合**
2. 简化模式要求**同时按住 Ctrl+Alt**
3. 只按 Ctrl 或只按 Alt 不会触发简化模式
4. 释放 Ctrl 或 Alt 后，简化模式失效，需要重新按完整组合

## 预期行为

根据需求，预期行为如下：

| 操作 | 是否触发 | 说明 |
|------|---------|------|
| Ctrl+Alt+1 | ✅ | 完整组合触发 |
| 保持 Ctrl+Alt，按 2 | ✅ | 简化模式触发 |
| 保持 Ctrl+Alt，按 3 | ✅ | 简化模式触发 |
| 只按 Ctrl+1 | ❌ | 缺少 Alt 修饰键 |
| 只按 Alt+1 | ❌ | 缺少 Ctrl 修饰键 |
| 释放所有键后，只按 1 | ❌ | 需要重新按完整组合 |
